# 植愈日记 工程架构说明

## 1. monorepo 分层原则

- apps 只承载可运行应用，不放跨端复用契约。
- packages 只承载共享类型、同步协议和配置说明，不承载运行时状态。
- 后端优先采用单体模块化，避免个人开发阶段过早拆分微服务。
- 前端严格双端适配，平台差异通过 platform 目录与条件编译承接。

## 2. 后端模块边界

- health：健康检查与基础可观测性。
- ai-proxy：统一代理 AI 请求，屏蔽密钥，做限流与审计。
- image：图片压缩、尺寸校验与上传前处理。
- sync：本地 SQLite 与云端 MySQL 的同步协议、版本号与冲突处理。
- scheduler：天气预警、养护提醒、推送编排。

## 3. 前端目录边界

- pages：页面级入口。
- components：可复用展示组件。
- services：请求封装、缓存编排、同步调度调用。
- platform：按平台拆分的适配器，专门处理 H5 与微信小程序差异。
- types：前端本地补充类型，优先复用 contracts。

## 4. 当前技术约束

- TypeScript 严格模式默认开启，禁止 any。
- 样式仅允许 UnoCSS，禁止自定义 CSS、SCSS、LESS。
- 前端缓存、请求、弹窗、路由统一走 Unibest 原生能力。
- 所有隐私数据默认按加密存储设计，AI 密钥仅后端保管。
- 数据流必须支持本地持久化、云端备份、回滚恢复。

## 5. 初始化顺序

1. 前端接入 Unibest 官方脚手架，并校准路径到 apps/client。
2. 后端接入 NestJS 官方脚手架，并保留当前模块边界。
3. 优先定义 contracts 包中的实体与同步协议，再写接口与页面。
