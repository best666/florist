# 组件库管理协议

## 触发条件（满足任一）

- 扫描代码时发现 ≥3 处重复的相同逻辑
- 用户明确要求"把这个抽成公共的"
- 重构阶段主动识别

**不触发**：日常业务开发、仅使用一次的代码。

## 执行协议

### Step 1: 识别

扫描时关注以下模式：
- UI 片段（相同的 DOM 结构 + 样式）
- 工具函数（相同的处理逻辑）
- composable（相同的状态管理逻辑）
- 类型定义（相同的 interface/type）

### Step 2: 提议

输出建议格式：

```
发现 N 处重复逻辑：

位置:
- [文件A:行号]
- [文件B:行号]
- [文件C:行号]

建议抽取为:
- 组件: apps/client/src/components/[Name].vue
- 或: apps/server/src/common/[name].ts

是否抽取？
```

### Step 3: 抽取 + 文档

用户确认后抽取，输出：

```
## 新增组件: [Name]

**文件**: [路径:行号]

**Props / 参数**:
| 名称 | 类型 | 必填 | 说明 |
|---|---|---|---|
| ... | ... | ... | ... |

**使用示例**:
```vue
<Name prop="value" />
```

**接入注意**: [边界情况 / 依赖]
```

### Step 4: 替换引用

抽取后将所有引用点替换为公共组件，每个替换点分别验证。

## 禁止

- 仅使用 1 次的逻辑不抽取（CLAUDE.md Simplicity First）
- 业务耦合度高不强行抽象
- 不创建"将来可能用"的空壳组件
