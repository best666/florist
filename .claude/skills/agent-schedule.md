# Agent 智能调度

## 触发条件

**仅以下场景启用**，单文件开发不触发：

- ≥3 个文件需要修改
- 跨项目/跨模块改造
- 批量脚本执行（如批量重命名、批量迁移）

## 调度策略

自动将任务拆分为三阶段流水线：

```
读文件（Explore Agent）→ 改代码（Edit/Write）→ 自动化校验（typecheck + build）
```

### 阶段说明

1. **读文件阶段** — 并行读取所有相关文件，建立完整的上下文视图。
2. **改代码阶段** — 按依赖关系排序修改顺序，避免冲突。独立文件的修改可并行。
3. **校验阶段** — 修改完成后自动执行类型检查和编译验证，确保无回归。

## 预校验

- 执行前校验文件路径是否存在、命令是否可用。
- 对批量操作先做 dry-run 预览，确认无误后再正式执行。

## 约束

- 单文件修改不触发本规则，避免过度工程化。
- 任务完成后输出各 Agent 的执行摘要和耗时统计。
