# 通用调试

## 执行标准

1. **定位根因** — 从现象追踪到根本原因，不满足于表面修复。优先阅读报错日志、链路追踪、相关源码。
2. **最小范围修改** — 仅改动与问题直接相关的代码，禁止新增无关功能、修改非目标文件。
3. **输出规范** — 每次修复完成后输出：
   - 根因分析（为什么出问题）
   - 代码 diff（修改了什么）
   - 落地验证步骤（如何确认修复有效）

## 编码规范

- 优先复用项目现有组件和工具函数，参考 [CLAUDE.md](../../CLAUDE.md) 中的架构说明。
- 当修改涉及 ≥3 个文件时，可启用 Agent 并行处理以提升效率。
- 修复后自动运行 `pnpm typecheck` 验证类型安全。

## 高频问题归集

- 调试过程中识别到的重复性问题，标记类型与触发条件，便于后续归集为快捷指令。
- 在修复完成后，若该问题模式可能再次出现，主动提示用户是否需要归入快捷指令库。
