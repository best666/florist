# 批量修复

## 适用场景

- 多项目同步 BUG 修复
- 单项目多文件统一整改（如 API 变更、类型迁移、命名规范统一）
- **不改动项目独有业务逻辑** — 仅做模式一致的批量替换或对齐。

## 执行流程

1. **范围确认** — 先列出所有需要修改的文件清单，请用户确认后再执行。
2. **分批处理** — 大批量任务启用 Agent 自动遍历目录，按模块分批提交。
3. **公共逻辑识别** — 处理过程中若发现 ≥3 处重复的通用逻辑，主动提出抽取为公共组件/工具函数的建议。

## 输出规范

修改完成后输出三份文档：

1. **修改文件清单** — 每个文件的路径、修改行数、变更类型
2. **全量 diff** — 所有文件的完整差异汇总
3. **修复汇总报告** — 包含：
   - 修复背景与目标
   - 涉及模块/文件统计
   - 潜在风险点（如影响面较大的变更）
   - 建议的回归验证范围
